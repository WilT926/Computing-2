(static) axialToPixel(q, r) → {Object}

Description:

  • Converts axial hex grid coordinates (q, r) into geometric (x, y) pixel/SVG unit coordinates. These coordinates represent the center of a hexagon in a "native" grid system, before any board-wide centering offsets are applied. The calculation is based on the true config.HEX_SIZE. This function is fundamental for the hex grid layout, translating abstract grid addresses into drawable positions.

(static) createInteractiveElements(hexagonsData) → {void}

Description:

  • Creates and renders the clickable SVG elements for settlement points (as circles at vertices) and road lines (as lines along edges). It processes the hexagonsData (from renderBoard) to identify unique vertex and edge positions on the grid. To ensure that settlement circles appear visually on top of road lines if they overlap, all road lines are created and appended to the SVG first, followed by all settlement circles. This function adds the (by default invisible) but clickable interaction layer to the board for player actions.

(static) defineHexagonClipPath() → {void}

Description:

  • Creates and defines a reusable SVG element with a hexagonal shape. This clip path is stored in the SVG's section and is identified by id="hexagon-clip". It is used to clip the rectangular tile images into a hexagonal form, matching the visual size of the tiles to create spacing. The function ensures that this clip path is defined only once, even if called multiple times. It relies on config.svgBoardElement, config.HEX_SIZE, and config.HEX_VISUAL_OFFSET.

(static) generateTiles() → {Array.<Object>}

Description:

  • Creates an array of tile data objects based on the definitions and counts specified in the config.tileTypes object. (static) getHexagonVerticesSVG(svgCenterPos) → {Array.<{x: number, y: number}>}

Description:

  • Calculates the SVG coordinates of the 6 vertices (corners) of a single hexagon, given its final center position on the SVG canvas. These calculations are based on the true config.HEX_SIZE, ensuring that interactive elements (like settlement points) are placed accurately at the true grid intersections, rather than on the potentially shrunk visual edges of the hexagons. This provides precise corner locations needed for placing settlement points and defining the endpoints for road lines.

(static) initGame() → {void}

Description:

  • The main initialization function for the Catan board. It orchestrates the entire setup process by calling various helper functions in the correct sequence. It first generates the standard set of Catan tiles, shuffles them, then calls renderBoard to draw the visual board on the SVG canvas. Finally, if the board rendering is successful, it calls createInteractiveElements to add the clickable points and lines. This function is typically called once the HTML document is fully loaded and parsed (on the DOMContentLoaded event).

(static) renderBoard(shuffledTiles) → {Array.<Object>}

Description:

  • Renders the entire Catan board by drawing all shuffled game tiles as SVG elements. Each tile is constructed as an SVG group () containing a visible hexagonal polygon (for the outline and fallback color) and a clipped hexagonal image. The function also calculates the necessary offsets to ensure the entire board is centered within the SVG's defined viewBox. This is the primary visual rendering function for the game tiles.

(static) shuffleTiles(tiles) → {Array.<Object>}

Description:

  • Randomly shuffles the order of elements in a given array of tiles. This function uses the Ramda library's R.sortBy method, providing a randomizing callback function () => Math.random() as the sorting criterion. This effectively shuffles the array. This randomization is crucial for the replayability of Catan, ensuring a different board layout each game.
